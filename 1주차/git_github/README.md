## ⚠️ 이 자료에서 안 다루는 것

부트캠프 4주 프로젝트에 **거의 안 쓰이거나, 초보자에게 위험한** 명령어는 권장 동선에 등장하지 않습니다.

- `git push --force` / `git push -f` — 팀원 작업을 날릴 수 있어요. 절대 ❌
- `git reset --hard` — 로컬 변경사항이 복구 불가능하게 사라집니다
- `git rebase -i` (interactive rebase) — 히스토리 재작성, 초보자에게는 위험
- `git cherry-pick`, `git reflog`, `detached HEAD` — 필요한 상황은 멘토/팀에 물어보세요

위 명령어가 필요하다고 느껴지면 **3-02 안전한 되돌리기**의 라우팅 박스를 먼저 펴보세요.
