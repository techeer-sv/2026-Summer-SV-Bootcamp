# 12. Monitoring — 핸즈온 실습

## 초급: 내 서비스에 로깅 추가하고 직접 확인하기

**목표**: 애플리케이션 로깅과 업타임 모니터링의 기본 개념을 실제로 적용해본다.

**진행 순서**
1. 배포된 백엔드 API에 요청 로깅 미들웨어 추가 (요청 경로, 시간, 응답 코드 기록)
2. 일부러 에러를 발생시키는 요청을 보내고 로그에서 에러 내용 확인
3. 무료 업타임 모니터링 서비스(예: UptimeRobot)에 배포 URL 등록
4. 서버를 잠시 중지시켜 알림(이메일/슬랙 등)이 오는지 확인

**결과물**
- 정상/에러 요청에 대한 로그 캡처
- 업타임 모니터링 알림 캡처

## 고급: 에러 트래킹 + 대시보드 구축

**목표**: 에러를 자동으로 수집하고, 구조화된 로그와 메트릭 대시보드로 서비스 상태를 한눈에 파악할 수 있도록 구성한다.

**진행 순서**
1. Sentry(또는 유사 도구)를 백엔드/프런트엔드에 연동
2. 일부러 에러를 발생시켜 Sentry에 수집되는지 확인하고 스택트레이스 분석
3. 기존 로그를 로그레벨(info/warn/error)과 JSON 포맷을 갖춘 구조화된 로깅으로 전환
4. Grafana/Prometheus 또는 무료 대시보드 도구로 응답시간/요청수를 시각화
5. 특정 에러율을 초과하면 알림이 가도록 규칙 설정

**결과물**
- 실시간 에러 확인이 가능한 Sentry 대시보드
- 응답시간/요청수를 보여주는 간단한 메트릭 대시보드
