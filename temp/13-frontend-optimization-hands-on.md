# 13. 프런트엔드 최적화 기술 — 핸즈온 실습

## 초급: Lighthouse로 내 페이지 성능 진단하기

**목표**: 성능 측정 도구를 사용해 점수를 확인하고, 기본적인 최적화로 점수를 개선하는 경험을 한다.

**진행 순서**
1. (세션 4에서 만든) 페이지를 Lighthouse로 성능 점수 측정
2. 이미지 용량을 줄이고(압축/WebP 변환 등) 재측정
3. CSS/JS 파일에 minify 적용 후 재측정
4. 점수 변화를 표로 정리하고, 각 변화의 원인을 분석

**결과물**
- 최적화 전/후 Lighthouse 리포트 비교 캡처

## 고급: React 앱 Core Web Vitals 개선하기

**목표**: 실제 SPA의 Core Web Vitals를 측정하고, 코드 스플리팅·번들 최적화·렌더링 전략 변경으로 개선한다.

**진행 순서**
1. (세션 4 고급 실습의) React 앱에서 LCP, INP, CLS 측정
2. `React.lazy` + dynamic import로 코드 스플리팅 적용
3. 번들 분석 도구로 용량이 큰 의존성을 찾아 더 가벼운 대안으로 교체
4. 이미지 lazy loading 및 적절한 크기/포맷 적용
5. (선택) 일부 페이지를 Next.js로 SSR/SSG 전환 후 기존 CSR 버전과 비교
6. 개선 전/후 지표를 표로 정리

**결과물**
- Core Web Vitals 개선 보고서 (Before/After 수치 포함)
