# 14. 고급 AI 관련 세션 — 핸즈온 실습

## 초급: AI API로 텍스트 요약 기능 만들기

**목표**: LLM API를 직접 호출해보고, 프롬프트에 따라 결과가 달라지는 것과 토큰/비용/rate limit 개념을 체감한다.

**진행 순서**
1. Claude 또는 OpenAI API 키 발급 및 환경변수로 설정
2. 백엔드 엔드포인트 작성: 사용자가 보낸 긴 텍스트를 API에 전달해 요약 결과 반환
3. 프롬프트를 바꿔가며 결과 비교 (예: "한 줄 요약" vs "핵심 포인트 3가지")
4. 응답에 포함된 토큰 사용량을 확인하고 대략적인 비용 계산
5. 짧은 시간에 여러 번 요청해 rate limit 에러를 발생시키고, 적절한 에러 처리 추가

**결과물**
- 텍스트 요약 API 엔드포인트
- 프롬프트별 결과 비교 및 토큰/비용 정리 노트

## 고급: 내 프로젝트 데이터 기반 RAG 챗봇 만들기

**목표**: 임베딩, 벡터 검색, RAG, function calling을 실제 프로젝트 데이터에 적용해본다.

**진행 순서**
1. 기존 프로젝트(예: 게시판)의 데이터를 문서 형태로 정리
2. 임베딩 모델로 각 문서를 벡터로 변환, 벡터DB(또는 간단한 in-memory 벡터 스토어)에 저장
3. 사용자 질문을 임베딩하여 유사한 문서를 검색 (retrieval)
4. 검색된 문서를 컨텍스트로 LLM에 전달해 답변 생성 (RAG)
5. "특정 게시글 검색"과 같은 도구를 LLM이 호출하도록 function calling 구현
6. 비용/지연시간을 측정하고, 잘못된 답변을 줄이기 위한 가드레일(프롬프트 제한 등) 추가

**결과물**
- "내 프로젝트에 대해 물어보면 답해주는" RAG 챗봇 데모
- 비용/지연시간 측정 및 가드레일 적용 내용 정리
