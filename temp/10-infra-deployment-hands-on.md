# 10. 인프라 구축 및 배포 — 핸즈온 실습

## 초급: 정적 포트폴리오 사이트 배포하기

**목표**: 로컬 코드를 실제로 인터넷에 배포하고, 코드 수정이 자동으로 반영되는 흐름을 경험한다.

**진행 순서**
1. (세션 4에서 만든) 정적 페이지를 GitHub repository에 push
2. Vercel 또는 Netlify에 가입 후 해당 repo 연결
3. 자동 배포 완료 후 제공된 URL로 접속 확인
4. 페이지 내용을 수정해서 다시 push → 자동으로 배포가 갱신되는지 확인
5. (선택) 무료 도메인 또는 제공된 서브도메인 연결

**결과물**
- 누구나 접속 가능한 배포 URL
- 코드 수정 → 자동 재배포 과정 캡처

## 고급: 풀스택 앱 클라우드 배포

**목표**: 프런트엔드, 백엔드, DB로 구성된 서비스를 클라우드에 배포하고 도메인/HTTPS까지 연결한다.

**진행 순서**
1. (세션 5/6에서 만든) 백엔드 API를 PaaS(Render, Railway 등)에 배포
2. DB 접속정보, 비밀키 등을 클라우드 대시보드의 환경변수로 설정
3. 프런트엔드를 Vercel에 배포하고, 배포된 백엔드 API URL로 연결
4. CORS 에러가 발생하면 원인을 파악하고 백엔드 설정으로 해결
5. 커스텀 도메인을 연결하고 HTTPS(자동 인증서)가 적용되는지 확인
6. (선택) VM에 직접 Nginx를 설치해 리버스 프록시를 구성하는 심화 실습 진행

**결과물**
- 프런트엔드+백엔드+DB가 모두 클라우드에서 동작하는 서비스 URL
- CORS 에러 발생 → 해결 과정 메모
