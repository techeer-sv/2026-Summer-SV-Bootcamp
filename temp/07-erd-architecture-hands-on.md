# 7. ERD & 시스템 아키텍쳐 — 핸즈온 실습

## 초급: 블로그 서비스 ERD 그려보기

**목표**: 엔티티-속성-관계 개념을 이해하고, 간단한 서비스의 데이터 구조와 시스템 구조를 다이어그램으로 표현한다.

**진행 순서**
1. 요구사항 정리: "사용자는 게시글을 작성하고, 게시글에는 여러 댓글이 달릴 수 있다"
2. 엔티티 도출: `User`, `Post`, `Comment`
3. 각 엔티티의 속성(컬럼) 정의 (예: User - id, name, email)
4. 관계 정의: User-Post (1:N), Post-Comment (1:N)
5. ERDCloud(또는 draw.io)로 ERD 작성
6. 클라이언트-서버-DB로 구성된 기본 시스템 아키텍처 다이어그램 작성

**결과물**
- ERDCloud로 작성한 블로그 ERD
- 클라이언트-서버-DB 아키텍처 다이어그램 1장

## 고급: 쇼핑몰 서비스 ERD + 아키텍처 설계

**목표**: N:N 관계 처리, 정규화, 확장 가능한 시스템 아키텍처 설계까지 경험한다.

**진행 순서**
1. 요구사항 정리: 사용자, 상품, 주문, 주문상세, 카테고리, 리뷰
2. N:N 관계(상품-카테고리, 주문-상품)를 중간 테이블로 풀어서 설계
3. 정규화 적용 전/후 테이블 구조를 각각 그려보고 비교 (3NF까지)
4. 외래키 및 제약조건(UNIQUE, NOT NULL, ON DELETE CASCADE 등) 정의
5. 모놀리식 아키텍처 다이어그램과 마이크로서비스(주문/상품/유저 서비스 분리) 아키텍처 다이어그램을 각각 작성
6. 캐시 계층, 로드밸런서를 추가했을 때 아키텍처가 어떻게 바뀌는지 표현

**결과물**
- 정규화 전/후 ERD 비교본
- 모놀리식 vs MSA 아키텍처 다이어그램 2종
