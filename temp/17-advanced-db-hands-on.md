# 17. Advanced DB — 핸즈온 실습

## 초급: 느린 쿼리 체감하고 인덱스 효과 확인하기

**목표**: 데이터가 많아질 때 쿼리가 느려지는 이유와 인덱스의 효과를 직접 측정하며 이해한다.

**진행 순서**
1. 스크립트로 10만 건 이상의 더미 데이터를 테이블에 삽입
2. 인덱스가 없는 컬럼으로 검색 쿼리를 실행하고 소요 시간 측정
3. 해당 컬럼에 인덱스를 생성한 뒤 동일 쿼리를 다시 실행하고 시간 비교
4. `EXPLAIN`으로 인덱스 적용 전/후 실행계획이 어떻게 달라졌는지 확인

**결과물**
- 인덱스 적용 전/후 쿼리 시간 비교
- `EXPLAIN` 결과 비교 캡처

## 고급: 트랜잭션 격리수준 & 복제 구조 실습

**목표**: 동시성 문제(락/데드락), 격리수준의 차이, 복제 구조를 직접 구성하고 관찰한다.

**진행 순서**
1. 두 개의 DB 커넥션으로 동시에 같은 row를 수정하는 상황을 만들어 락/데드락 재현
2. 격리수준(READ COMMITTED, REPEATABLE READ 등)을 바꿔가며 동작 차이 관찰
3. Docker로 Primary-Replica 복제 구조 구성 (MySQL/Postgres replication)
4. Primary에 데이터를 쓰고 Replica에서 읽으며 복제 지연(replication lag) 관찰
5. (선택) 테이블을 특정 기준으로 파티셔닝하고 쿼리 성능 비교

**결과물**
- 데드락 재현 로그
- 격리수준별 동작 비교 메모
- Primary-Replica 복제 구성 캡처
