## 2. Git, GitHub, & 프로젝트 구조
- 버전 관리의 중요성
    - 버전 관리의 이점
    - 작업한 결과물에 대해서 예전 버전으로 돌아가기 위해 어떻게 할 것인가
    - 손쉽게 버전을 관리할수있다
- Git & GitHub 개요
    - Git?
        - 프로젝트의 변경 사항을 기록하고 여러명이 효율적으로 협엽할 수 있게 돕는 분산 버전 관리 시스템
        - 코드의 변경이력을 기록하고 같은 파일의 버전을 관리하고 보장
    - GitHub?
        - Git을 기반으로 하는 클라우드 방식의 코드 저장소이자 개발자들의 소셜 네트워크 서비스
- 설치 & 환경설정
    - git 설치 방법
    - 환경설정 (git config --global config user.name, git config --global config user.email)
- Git 기본 워크플로우
    - repository 만들기
    - repository clone
    - git clone vs git init
- 실습
    - git add
    - git commit
    - git status
    - .gitignore
- 브랜치란?
    - 원본 코드는 그래로 두고 나만의 복사본 세계에서 안전하게 코드를 고칠수 있게 해주는 기능
    - 브랜치 명령어
- 개발 전략 with Git
    - 브랜치 관리
        - main, master, develop, etc
    - 커밋 메시지 convention
- 프로젝트 관리
    - issue 생성
    - Pull request
    - draft PR
