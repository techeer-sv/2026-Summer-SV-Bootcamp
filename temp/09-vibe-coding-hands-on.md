# 9. 바이브 코딩 기초 세션 — 핸즈온 실습

## 초급: AI와 함께 텍스트 게임 만들기

**목표**: AI 코딩 도구에게 자연어로 요구사항을 전달하고, 결과를 실행/확장/디버깅하는 기본 루프를 경험한다.

**진행 순서**
1. ChatGPT/Claude 등에게 "숫자 맞추기 게임"을 만들어달라고 요청
2. 생성된 코드를 실행해보고 동작 확인
3. "힌트 기능을 추가해줘", "시도 횟수를 5번으로 제한해줘" 등 추가 요청으로 기능 확장
4. 코드 중 이해되지 않는 부분을 AI에게 "이 코드가 왜 이렇게 동작하는지 설명해줘"라고 질문
5. 코드를 일부러 잘못 수정한 뒤, 에러 메시지를 그대로 AI에게 보여주고 디버깅 요청

**결과물**
- 실행되는 텍스트 게임
- "무엇을 어떻게 물어봤는지"를 정리한 프롬프트 기록 노트

---

## 고급: AI IDE로 기능 하나를 end-to-end 구현

**목표**: 실제 프로젝트에 새 기능을 추가하면서 프롬프트 작성 → 멀티파일 변경 리뷰 → 테스트 → 커밋까지의 흐름을 익힌다.

**진행 순서**
1. 기존 프로젝트(예: 세션 5의 Todo API)에 "마감일(due date) + 완료 여부 필터" 기능 추가를 목표로 설정
2. AI에게 입출력 예시와 영향받는 파일까지 포함해 구체적으로 요구사항 작성
3. AI가 변경한 여러 파일의 diff를 직접 리뷰하며 불필요하거나 잘못된 변경 찾아내기
4. AI에게 해당 기능의 테스트 코드 생성을 요청하고 직접 실행해 통과 여부 확인
5. AI에게 커밋 메시지 작성을 요청한 뒤, 직접 다듬어서 커밋
6. "좋았던 프롬프트 vs 안 좋았던 프롬프트"를 비교한 회고 노트 작성

**결과물**
- 기능이 추가된 PR (테스트 코드 포함)
- 프롬프트 작성 회고 노트
