# 11. CI/CD — 핸즈온 실습

## 초급: GitHub Actions로 자동 테스트 실행하기

**목표**: push할 때마다 자동으로 테스트가 실행되는 가장 단순한 CI 파이프라인을 구성한다.

**진행 순서**
1. 기존 프로젝트에 간단한 단위 테스트 1~2개 작성
2. `.github/workflows/ci.yml` 작성 — push 시 의존성 설치 후 테스트 실행
3. 일부러 실패하는 코드를 push해서 Actions 탭에서 빨간 X(실패) 확인
4. 코드를 고쳐서 다시 push → 초록 체크(성공) 확인

**결과물**
- GitHub Actions에 성공/실패 기록이 모두 남아있는 repository

## 고급: PR 검증 + 자동 배포 파이프라인 구축

**목표**: PR 단계의 자동 검증과, main 병합 시 자동 배포까지 이어지는 전체 파이프라인을 구성한다.

**진행 순서**
1. PR 생성 시에만 동작하는 워크플로우 작성 (lint + test)
2. main 브랜치에 merge되면 자동으로 배포되는 워크플로우 작성 (세션 10에서 구성한 배포 환경 활용)
3. 배포에 필요한 비밀값(API 키, 서버 접속정보 등)을 GitHub Secrets로 등록하고 워크플로우에서 사용
4. 의존성 캐싱을 적용해 빌드 시간 변화 비교
5. 배포를 일부러 실패시키고, 롤백 또는 재배포 과정을 직접 진행

**결과물**
- PR → 자동 lint/test → main 병합 → 자동 배포까지 이어지는 파이프라인
- 캐싱 적용 전/후 빌드 시간 비교
