# 16. 비동기 처리와 큐 — 핸즈온 실습

## 초급: 동기 vs 비동기 코드 비교 실험

**목표**: 동기 처리가 다른 요청을 막는 문제를 직접 경험하고, 비동기 처리로 해결되는 과정을 비교한다.

**진행 순서**
1. 시간이 걸리는 작업(예: 3초 대기)을 동기적으로 처리하는 코드 작성 → 그 사이 다른 요청이 막히는 것 확인
2. 같은 작업을 async/await로 바꿔서 다른 요청이 막히지 않는 것을 확인
3. "이메일 발송 시뮬레이션"을 콜백 / Promise / async-await 세 가지 방식으로 각각 작성해보고 비교

**결과물**
- 동기/비동기 방식의 응답 시간 비교 캡처
- 콜백/Promise/async-await 코드 비교 노트

## 고급: 백그라운드 작업 큐로 이미지 처리 파이프라인 구현

**목표**: 메시지 큐와 워커를 이용해 무거운 작업을 비동기로 처리하고, 실패 시 재시도까지 구현한다.

**진행 순서**
1. BullMQ + Redis(또는 유사 큐 라이브러리)로 작업 큐 환경 구성
2. 사용자가 이미지를 업로드하면 즉시 "처리중" 응답을 반환하고, 실제 리사이즈 작업은 큐에 등록 (producer)
3. 별도의 워커 프로세스가 큐에서 작업을 꺼내 이미지 리사이즈 수행 (consumer)
4. 작업 상태(대기/처리중/완료/실패)를 조회할 수 있는 API 추가
5. 일부러 작업을 실패시키고, 재시도(retry) 및 dead-letter queue 동작 확인

**결과물**
- 업로드 → 큐 등록 → 워커 처리 → 상태 조회까지 동작하는 파이프라인
- 재시도/실패 처리 동작 로그
