# 15. 백엔드 최적화 기술 - 캐시 — 핸즈온 실습

## 초급: 메모리 캐시로 API 응답 속도 비교하기

**목표**: 캐시가 응답 속도에 어떤 영향을 주는지 직접 측정하며 체감한다.

**진행 순서**
1. 의도적으로 느린 API 엔드포인트 작성 (예: 1초 delay를 주는 외부 호출 흉내)
2. 캐시 적용 전, 첫 요청과 두 번째 요청의 응답 시간 측정
3. 간단한 in-memory Map/객체로 캐시 구현 (TTL 포함)
4. 캐시 적용 후 응답 시간을 다시 측정하고 비교

**결과물**
- 캐시 적용 전/후 응답시간 비교 표

## 고급: Redis로 캐시 레이어 구축 + 무효화 전략

**목표**: Redis를 이용한 cache-aside 전략과 캐시 무효화, 부하 테스트를 통한 효과 측정을 경험한다.

**진행 순서**
1. Redis를 docker-compose에 추가
2. 자주 조회되는 DB 쿼리 결과를 cache-aside 방식으로 Redis에 캐싱
3. TTL을 설정하고 만료 후 자동 갱신되는지 확인
4. 데이터 수정 API 호출 시 관련 캐시를 무효화(invalidate)하는 로직 추가
5. 부하 테스트 도구(예: autocannon)로 캐시 적용 전/후 처리량(req/sec) 비교
6. (선택) 정적 자산에 CDN 캐싱 헤더를 설정해보기

**결과물**
- Redis 캐시 + 무효화 로직이 적용된 API
- 부하 테스트 결과 비교 (캐시 적용 전/후 req/sec)
