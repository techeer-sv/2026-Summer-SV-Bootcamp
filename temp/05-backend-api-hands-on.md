# 5. 백엔드 기초와 API 설계 — 핸즈온 실습

## 초급: Todo API 서버 만들기

**목표**: HTTP 메서드와 상태 코드를 이해하고, 메모리 기반 CRUD API를 직접 만들고 테스트한다.

**진행 순서**
1. Node.js + Express 프로젝트 초기 설정
2. 메모리 배열을 데이터 저장소로 사용하여 라우트 작성
   - `GET /todos`, `POST /todos`, `PUT /todos/:id`, `DELETE /todos/:id`
3. Postman/Thunder Client로 각 엔드포인트 요청 보내고 응답 확인
4. 존재하지 않는 id로 요청 시 `404` 응답 처리 추가
5. 잘못된 입력(빈 제목 등)에 `400` 응답 처리 추가
6. 200, 201, 400, 404, 500 상태 코드별 의미를 정리한 표 작성

**결과물**
- 실행 가능한 Todo API 서버
- 각 엔드포인트별 요청/응답 캡처가 포함된 Postman 컬렉션

## 고급: 인증이 포함된 게시판 API + 문서화

**목표**: 계층형 아키텍처, 인증/인가, 검증, API 문서화까지 포함한 실전형 REST API를 설계하고 구현한다.

**진행 순서**
1. 리소스 설계: `users`, `posts` (작성자-게시글 관계)
2. Controller-Service-Repository 계층으로 코드 구조 분리
3. 회원가입/로그인 구현 (비밀번호 해싱 + JWT 발급)
4. 인증 미들웨어 작성 — 로그인한 사용자만 글쓰기 가능, 본인 글만 수정/삭제 가능
5. 입력값 검증 라이브러리 적용 및 표준화된 에러 응답 포맷 설계 (예: `{ "error": { "code": ..., "message": ... } }`)
6. Swagger(OpenAPI)로 전체 API 문서 자동 생성

**결과물**
- 인증/인가가 적용된 게시판 CRUD API
- Swagger UI에서 모든 엔드포인트를 테스트할 수 있는 문서 페이지
