# 4. 프런트엔드 & UIUX 기초 — 핸즈온 실습

## 초급: 정적 프로필 카드 페이지 만들기

**목표**: HTML/CSS/JS의 기본 구성요소와 UI/UX 기본 개념을 활용해 정적인 프로필 페이지를 직접 완성한다.

**진행 순서**
1. Figma(또는 종이)에 간단한 프로필 카드 와이어프레임 그려보기 (사진, 이름, 한줄소개, 버튼)
2. HTML로 시맨틱 구조 작성 (`header`, `main`, `footer` 등)
3. CSS box model과 flexbox로 카드 레이아웃 배치 (가운데 정렬, 여백, 카드 그림자)
4. 색상/폰트를 적용하여 보기 좋은 스타일링 마무리
5. JS로 "다크모드 토글" 버튼 구현 (DOM 선택, 이벤트 리스너, 클래스 토글)
6. 브라우저 개발자도구를 열어 실시간으로 HTML/CSS를 수정해보며 결과 확인

**결과물**
- 다크모드 토글이 동작하는 정적 프로필 카드 페이지 (HTML/CSS/JS 파일)
- Figma 와이어프레임과 완성본 비교 캡처

## 고급: 공개 API 연동 React 검색 앱 만들기

**목표**: 컴포넌트 기반 아키텍처와 비동기 데이터 연동, 반응형/접근성 고려까지 포함한 SPA를 제작한다.

**진행 순서**
1. Vite로 React 프로젝트 생성
2. 공개 API 1개 선정 (예: 날씨, 영화, 포켓몬 등)
3. 컴포넌트 분리: `SearchBar`, `ResultList`, `ResultCard`
4. `useState`/`useEffect`로 검색어 입력에 따른 API 호출 및 상태 관리
5. 로딩 중 / 에러 / 결과 없음 상태에 대한 UI 분기 처리
6. Tailwind 등으로 모바일 퍼스트 반응형 레이아웃 적용
7. Lighthouse 접근성(a11y) 검사 → `alt` 속성, 명도 대비, 키보드 포커스 등 개선

**결과물**
- 검색 기능이 동작하는 API 연동 SPA
- 모바일/데스크탑 화면 캡처
- Lighthouse 접근성 점수 개선 전/후 비교
